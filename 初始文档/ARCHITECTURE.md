## 🔄 核心处理单元详解

### 单元1: BERT编码处理

```
输入: "Design an 8-bit binary counter"
      │
      ├─→ Tokenization (分词)
      │   tokens = ['Design', 'an', '8', '-', 'bit', 'binary', 'counter']
      │
      ├─→ Token IDs (转ID)
      │   ids = [2573, 1037, 1022, 1011, 2096, 3571, 8281]
      │
      ├─→ Token Embeddings (初始嵌入)
      │   每个token → 768维向量
      │
      ├─→ BERT Forward Pass
      │   通过12层Transformer
      │   └─ 每层self-attention学习token间关系
      │
      └─→ 输出: 
          句子向量 = [CLS] token的输出
          = [0.1, -0.2, 0.5, ..., 0.3]  (768维)

特点: 
  • 捕捉语义信息
  • 支持多语言
  • 经过大规模预训练
```

### 单元2: AST解析处理

```
输入: Verilog代码
      module counter(
        input clk,
        input rst_n,
        output [7:0] count
      );
        always @(posedge clk)
          if (!rst_n) count <= 0;
          else count <= count + 1;
      endmodule

      │
      ├─→ 词法分析
      │   识别token: MODULE, IDENTIFIER, ...
      │
      ├─→ 语法分析

      │

### 单元4: 深度学习推理

```
输入:
  req_vector [1, 768]     ─维度─ 需求语义向量
  code_vector [1, 768]    ─维度─ 代码语义向量
  alignment_matrix [2, 2] ─维度─ 节点连接矩阵
    [[1.0,  0.88],      (需求→代码连接强度)
     [0.88, 1.0]]       (代码→需求连接强度)

┌─ 步骤1: 特征投影
│  输入: [req_vec, code_vec] concat → [2, 768]
│  Linear(768 → 256) + ReLU
│  输出: [2, 256]
│
├─ 步骤2: GAT处理 (学习节点间关系)
│  Layer 1:
│    输入: [2, 256] + alignment_matrix
│    多头注意力处理
│    输出: [2, 256]
│    ReLU激活
│  Layer 2:
│    输入: [2, 256] + alignment_matrix
│    输出: [2, 128]
│
├─ 步骤3: Bi-GRU (学习序列上下文)
│  序列化: [2, 128] → [1, 2, 128]
│  前向GRU: 处理依赖关系
│  反向GRU: 处理影响关系

│
├─ 步骤4: 分类头
│  FC(64 → 128) + ReLU
│  Dropout(0.1)
│  FC(128 → 1) + Sigmoid
│  输出: 0.15 ∈ [0, 1]
│
└─ 输出: 不一致概率 = 0.15
