**函数**: `NLPSemanticExtractor.get_semantic_vector(text: str) -> np.ndarray`

**位置**: `src/semantic_extraction.py`, 第 1062 行

**详细操作步骤**:

```
get_semantic_vector(req_text) -> np.ndarray (768,)
  │
  ├─ [检查BERT可用性]
  │   └─ if not self.bert_available:
  │       └─ 返回基于TF-IDF的简单向量
  │
  ├─ [第1步：语法依赖分析]
  │   └─ dependency_info = self.analyze_syntax_dependencies(text)
  │       └─ 提取语法结构信息用于后续增强
  │
  ├─ [第2步：BERT编码]
  │   ├─ inputs = self.tokenizer(
  │   │     text,
  │   │     max_length=512,          # 最大512个token
  │   │     truncation=True,         # 超长截断
  │   │     padding=True,            # 填充到512
  │   │     return_tensors="pt"      # 返回PyTorch张量
  │   │   )
  │   └─ 返回 {
  │       'input_ids': [...],        # Token ID列表
  │       'attention_mask': [...],   # 注意力掩码
  │       'token_type_ids': [...]    # 段ID
  │     }
  │
  ├─ [第3步：转移到计算设备]
  │   └─ inputs = {k: v.to(self.device) for k, v in inputs.items()}
  │       ├─ 如果有GPU：转到GPU
  │       └─ 否则使用CPU
  │
  ├─ [第4步：执行BERT forward pass]
  │   ├─ with torch.no_grad():
  │   │     outputs = self.model(**inputs)
  │   └─ outputs 包含：
  │       ├─ last_hidden_state: (batch_size=1, seq_len=512, hidden_size=768)
  │       ├─ pooler_output: (batch_size=1, hidden_size=768)
  │       └─ 其他注意力分布等
  │
  ├─ [第5步：提取[CLS]向量作为句子向量]
  │   ├─ sentence_embedding = outputs.last_hidden_state[:, 0, :].cpu().numpy().squeeze()
  │   │   ├─ outputs.last_hidden_state: (1, 512, 768)
  │   │   ├─ [:, 0, :]: 取第一个token ([CLS]) -> (1, 768)
  │   │   ├─ .cpu().numpy(): 转为numpy -> (768,)
  │   │   └─ .squeeze(): 去除多余维度
  │   └─ sentence_embedding: np.ndarray, shape (768,)
  │
  ├─ [第6步：语法依赖增强 (可选)]
  │   └─ 如果dependency_info不为空：
  │       ├─ 使用语法结构信息加权调整向量
  │       ├─ 突出重要语法单元的影响
  │       └─ sentence_embedding = enhanced_embedding
  │
  ├─ [第7步：L2规范化]
  │   └─ sentence_embedding = sentence_embedding / np.linalg.norm(sentence_embedding)
  │       ├─ 确保向量模长为1
  │       └─ 便于后续的余弦相似度计算
  │
  └─ [第8步：返回]
      └─ return sentence_embedding  # np.ndarray, shape (768,)
```

    req_id,
    req_elements,
    req_vector,
    code_elements,
    code_vector,

  │       ├─ 公式: cos_sim = (A·B) / (||A|| * ||B||)
  │       ├─ A = req_vector (768,)
  │       ├─ B = code_vector (768,)
  │       ├─ 结果范围: [-1, 1]，通常在 [0, 1]
  │       └─ 返回: 一个浮点数，表示相似度分数
  │
  ├─ [第2步：提取NLP和代码关键字]
  │   ├─ req_keywords = req_elements.get('keywords', [])
  │   │   └─ 例: ['模块', '8比特', '计数器', '异步复位']
  │   └─ code_keywords = list(code_elements.get('keywords', {}).keys())
  │       └─ 例: ['reg', 'always', 'posedge', 'clk']
  │
  ├─ [第3步：使用语法库进行深层分析]
  │   ├─ nlp_patterns = self.nlp_syntax.extract_semantic_patterns(req_text)
  │   │   ├─ 提取语义模式：
  │   │   │   ├─ 主题-属性模式 (subject-attribute)
  │   │   │   ├─ 结构-功能模式 (structure-function)
  │   │   │   └─ 约束-需求模式 (constraint-requirement)
  │   │   └─ 返回: 模式字典
  │   │
  │   └─ code_constructs = self.code_syntax.extract_code_constructs(code_text)
  │       ├─ 提取代码构造：
  │       │   ├─ 顶层模块结构
  │       │   ├─ 组合逻辑块
  │       │   ├─ 时序逻辑块
  │       │   └─ 时序路径
  │       └─ 返回: 构造字典
  │
  ├─ [第4步：使用语义映射规则库进行双向对齐]
  │   ├─ semantic_mappings = self.semantic_mapping_lib.find_semantic_mappings(
  │   │     req_keywords, code_keywords
  │   │   )
  │   │   ├─ 在语义映射库中查找匹配的映射规则
  │   │   ├─ 例如: '异步复位' -> 'rst_n' (async reset signal)
  │   │   ├─ 例如: '8比特' -> 'reg [7:0]' (8-bit register)
  │   │   └─ 返回: 映射规则列表
  │   │
  │   └─ mapping_confidence = 计算所有映射的平均置信度
  │       ├─ for each rule in semantic_mappings:
  │       │     score = self.semantic_mapping_lib.calculate_mapping_confidence(rule)
  │       ├─ mapping_confidence = mean(scores)
  │       └─ 范围: [0, 1]
  │
  ├─ [第5步：原有规则库匹配 (保留兼容性)]
  │   └─ matched_rules = self.rules_library.find_matching_rules(
  │        req_keywords, code_keywords
  │      )
  │       ├─ 在规则库中查找经典对齐规则
  │       ├─ 例如: "频率" + "MHz" 规则
  │       ├─ 例如: "计数器" + "count_up" 规则

  │        similarity,
  │        matched_rules,
  │        semantic_mappings,
  │        mapping_confidence,
  │        nlp_patterns,
  │        code_constructs,
  │        ...
  │      )
  │       ├─ 综合分析：
  │       │   ├─ 向量相似度权重: 35%
  │       │   ├─ 规则匹配权重: 35%
  │       │   ├─ 模式识别权重: 30%
  │       │   └─ 总分 = w1*sim + w2*rule_score + w3*pattern_score
  │       │
  │       ├─ 判定对齐状态：
  │       │   ├─ PERFECT (总分 >= 0.9): 完美对齐
  │       │   ├─ GOOD (总分 >= 0.7): 良好对齐
  │       │   ├─ PARTIAL (总分 >= 0.4): 部分对齐
  │       │   └─ POOR (总分 < 0.4): 对齐差
  │       │
  │       └─ 返回: 状态、置信度、原因
  │
  ├─ [第7步：获取匹配的规则ID]
  │   └─ aligned_rule = matched_rules[0]["id"] if matched_rules else None
  │

  │   └─ alignment_pairs = self._generate_alignment_pairs(
  │        req_elements, code_elements, semantic_mappings
  │      )
  │       ├─ 对于每个语义映射，生成对齐对：
  │       │   {
  │       │     'req': '需求方关键词',
  │       │     'code': '代码方关键词',
  │       │     'type': '对齐类型',
  │       │     'confidence': 置信度分数
  │       │   }
  │       │
  │       ├─ 例如第一对：
  │       │   {'req': '8比特', 'code': '[7:0]', 'confidence': 0.95}
  │       │
  │       ├─ 例如第二对：

  │       └─ 返回: 对齐对列表，用于DL模型输入
  │
  └─ [第9步：返回结果]

## 🎓 关键概念解释

### 什么是"对齐对" (alignment_pairs)？

对齐对是深度学习模型的核心输入，用来表示需求中的某个概念和代码中对应实现之间的映射关系：

```python
{
    'req': '8位',           # 需求中的概念
    'code': '[7:0]',        # 代码中对应的实现
    'type': 'bitwidth',     # 对齐类型
    'confidence': 0.99      # 置信度 (0-1)
}
```

- 如果有很多高置信度的对齐对，说明需求和代码对齐良好
- 如果对齐对很少或置信度低，说明存在语义间隙（隐性不一致）

### 什么是"显性" vs "隐性"不一致？

| 类型     | 定义           | 检测方法                  | 示例                           |
| -------- | -------------- | ------------------------- | ------------------------------ |
| **显性** | 明确的规则违反 | 规则引擎 (13条规则)       | 代码缺少'clk'信号              |
| **隐性** | 语义上的间隙   | 深度学习模型 (GAT+Bi-GRU) | 需求要求100MHz但代码无时序约束 |
